# 22. AgentMemory - 프로젝트 컨텍스트 기억

## 배경

`/clear` 명령으로 Session이 초기화되면 대화 이력뿐 아니라 프로젝트에 대해 설명했던 정보도 모두 사라진다.

Session(단기 기억)과 Memory(장기 기억)를 분리하여, 프로젝트 컨텍스트는 Session과 무관하게 영구 유지한다.

```
Session (칠판)  : 수업 끝나면 지운다. /clear 시 초기화 OK
Memory  (수첩)  : 중요한 건 여기 적어둔다. /clear해도 유지
```

## 구현 목표

프로젝트 루트에 `KODING.md` 파일을 장기 기억소로 사용한다.

```
koogAgent_H/          ← 프로젝트 루트
├── KODING.md         ← 장기 기억 (Session 무관)
└── .koogagent/
    └── projects/{projectDir}/{sessionId}/
        └── session.jsonl   ← 단기 기억 (Session별)
```

### Step 2: KodingMemoryStorage 구현 (TDD)

**테스트 파일**: `src/test/java/koogagent/storage/KodingMemoryStorageTest.java`

테스트 케이스:
- `getMemory_returnsNull_whenFileDoesNotExist()`
- `addMemory_createsFile_withListFormat()`
- `addMemory_appendsToExistingMemory()`
- `getMemory_returnsNull_whenFileIsBlank()`

**구현 파일**: `src/main/java/koogagent/storage/KodingMemoryStorage.java`

```java
package koogagent.storage;

import java.io.IOException;
import java.nio.file.Files;
import java.nio.file.Path;
import java.nio.file.StandardOpenOption;

public class KodingMemoryStorage implements AgentMemoryStorage {

    private final Path memoryFile;

    public KodingMemoryStorage() {
        this(Path.of("./KODING.md"));
    }

    public KodingMemoryStorage(Path memoryFile) {  // 테스트용 생성자
        this.memoryFile = memoryFile;
    }

    @Override
    public void addMemory(String content) throws IOException {
        String existing = getMemory();
        String line = "- " + content;
        String updated = (existing == null || existing.isBlank())
            ? line
            : existing + "\n" + line;
        Files.writeString(memoryFile, updated, StandardOpenOption.CREATE,
            StandardOpenOption.TRUNCATE_EXISTING);
    }

    @Override
    public String getMemory() throws IOException {
        if (!Files.exists(memoryFile)) return null;
        String content = Files.readString(memoryFile);
        return content.isBlank() ? null : content;
    }
}
```

저장 형식:
```
- 이 프로젝트는 Java + Maven 기반이야
- 테스트는 JUnit5를 써
- 코딩 스타일은 camelCase야
```

### Step 4: buildSystemPromptWithHistory 시스템 프롬프트 순서 변경

**파일**: `src/main/java/koogagent/CodingAgent.java`

시스템 프롬프트 구성 순서 (LLM이 앞부분을 더 중요하게 처리):

```
1. System Prompt   - 기본 지침
2. Project Memory  - KODING.md 내용 (항상 유지, /clear 무관)
3. Conversation Summary - 이전 대화 요약
4. Recent Conversation  - 최근 대화 (원본)
```

```java
static String buildSystemPromptWithHistory(List<Message> history, String summary, String memory) {
    if (summary == null && history.isEmpty() && memory == null) return SYSTEM_PROMPT;

    StringBuilder sb = new StringBuilder();
    sb.append("# System Prompt\n").append(SYSTEM_PROMPT);

    if (memory != null) {
        sb.append("\n\n# Project Memory\n");
        sb.append("아래는 이 프로젝트에 대해 기억해야 할 정보입니다:\n");
        sb.append(memory);
    }

    if (summary != null) {
        sb.append("\n\n# Previous Conversation Summary\n").append(summary);
    }

    if (!history.isEmpty()) {
        sb.append("\n\n# Recent Conversation\n");
        MessageFormatter.appendMessages(sb, history);
    }

    return sb.toString();
}
```

**테스트 파일**: `src/test/java/koogagent/CodingAgentHistoryTest.java` 기존 테스트 수정 + 신규 추가

신규 테스트 케이스:
- `buildSystemPromptWithHistory_withMemory_memoryBeforeSummary()` - Memory가 Summary보다 앞에 위치
- `buildSystemPromptWithHistory_withMemoryOnly()` - Memory만 있을 때
- `buildSystemPromptWithHistory_noHistoryNoSummaryNoMemory()` - 모두 null이면 기본 SYSTEM_PROMPT 반환
