# /clear 명령으로 Session 초기화 기능 만들기

## 배경

대화 이력이 누적되면 두 가지 문제가 생긴다.

1. **맥락 오염**: 이전 대화가 현재 대화에 영향을 주어 관련 없는 내용이 섞임
2. **비용 증가**: 매 요청마다 전체 이력을 전송하므로 대화가 길수록 토큰 비용 증가

## 초기화 방식: Session ID 변경

파일 삭제 대신 `CodingAgent`를 새로 생성한다.

| 항목 | 파일 삭제 | Session ID 변경 (채택) |
|------|-----------|----------------------|
| 이전 대화 | 완전 삭제 | 보존 |
| 구현 복잡도 | clear() 메서드 필요 | Agent 재생성만 |
| 디스크 사용량 | 적음 | 계속 증가 (텍스트라 크지 않음) |
| 이전 대화 참조 | 불가 | 가능 |

`CodingAgent(client, bashPath)` 를 호출하면 새 UUID가 자동 생성되고, 새 세션 디렉토리 아래에 빈 `session.jsonl`이 만들어진다. 이전 세션 디렉토리는 그대로 남는다.

```
.koogagent/projects/default/
├── 550e8400-e29b-41d4-a716-.../ ← 이전 세션 (보존)
│   └── session.jsonl
└── 7c9e6679-7425-40de-944b-.../ ← /clear 후 새 세션
    └── session.jsonl
```

## 구현 전략

변경 파일은 `KoogAgentMain.java` 하나다.  
`CodingAgent.java`와 `ConversationHistoryStorage` 계층은 수정하지 않는다.

## 구현 절차

### Step 1: try-with-resources → 수동 리소스 관리로 변경

`/clear` 시 agent를 재할당해야 하므로 Java의 try-with-resources 패턴을 제거한다.

```java
// Before
try (CodingAgent agent = new CodingAgent(client, bashPath)) { ... }

// After
CodingAgent agent = new CodingAgent(client, bashPath);
try {
    // 메인 루프
} finally {
    agent.close();
}
```

### Step 2: 시작 메시지에 /clear 안내 추가

```java
System.out.println("Coding Agent가 시작되었습니다.");
System.out.println("명령어: /clear (새 대화 시작), exit (종료)");
System.out.println();
```

### Step 3: /clear 명령 처리 추가

루프 안에서 `exit` 처리 이후에 추가한다.

```java
if (input.equals("/clear")) {
    agent.close();
    agent = new CodingAgent(client, bashPath); // 새 UUID로 새 세션
    System.out.println("새로운 대화가 시작되었습니다.");
    System.out.println();
    continue;
}
```

## 완성 후 Main 루프 구조

```java
CodingAgent agent = new CodingAgent(client, bashPath);
try {
    System.out.println("Coding Agent가 시작되었습니다.");
    System.out.println("명령어: /clear (새 대화 시작), exit (종료)");
    System.out.println();

    Scanner scanner = new Scanner(System.in);
    while (true) {
        System.out.print("User: ");
        if (!scanner.hasNextLine()) break;
        String input = scanner.nextLine().trim();

        if (input.isEmpty()) continue;

        if (input.equalsIgnoreCase("exit")) {
            System.out.println("종료합니다.");
            break;
        }

        if (input.equals("/clear")) {
            agent.close();
            agent = new CodingAgent(client, bashPath);
            System.out.println("새로운 대화가 시작되었습니다.");
            System.out.println();
            continue;
        }

        try {
            String response = agent.chat(input);
            System.out.println("Agent: " + response);
        } catch (Exception e) {
            System.out.println("오류: " + e);
        }
        System.out.println();
    }
} finally {
    agent.close();
}
```

## 검증 시나리오

1. 일반 대화 후 `/clear` 입력 → "새로운 대화가 시작되었습니다." 출력
2. `/clear` 후 이전 대화 내용 질문 → Agent가 모른다고 응답
3. `.koogagent/projects/default/` 디렉토리 확인 → 세션 디렉토리 2개 존재
4. 이전 세션 `session.jsonl` 확인 → 대화 이력 보존 여부 확인
