# 25. Claude Code SessionEnd 훅 - 작업 일지 자동화

## 목적

Claude Code 종료 시 오늘 어떤 작업을 처리했는지 자동으로 정리한다.

---

## Claude Code 훅 종류

| Hook | 발동 시점 |
|------|----------|
| `PreToolUse` | 도구 실행 직전 |
| `PostToolUse` | 도구 실행 완료 후 |
| `Stop` | Claude가 매 응답 턴을 마칠 때마다 |
| `SessionEnd` | 세션 종료 시 (exit, /clear, 창 닫기 등) |
| `Notification` | Claude가 알림을 보낼 때 |
| `SubagentStop` | 서브에이전트 종료 시 |

**작업 일지 목적에는 `SessionEnd` 사용** — 세션이 끝날 때 한 번만 실행.

---

## 훅 스크립트가 받는 데이터 (stdin JSON)

```json
{
  "session_id": "abc123",
  "transcript_path": "C:/Users/.../.claude/projects/.../transcript.jsonl",
  "cwd": "D:/sandbox/koogAgent_H",
  "hook_event_name": "SessionEnd"
}
```

`transcript_path` — 해당 세션의 전체 대화 내용이 JSONL 형식으로 저장된 파일 경로.

---

## 설치된 구성

### 훅 스크립트
**경로**: `C:\Users\EliteDesk\.claude\hooks\session-summary.ps1`

동작:
1. stdin에서 `transcript_path`, `cwd`, `session_id` 파싱
2. `transcript.jsonl`을 읽어 사용자 요청 목록 추출
3. 도구 사용 통계 집계 (`Edit`, `Bash`, `Read` 등 호출 횟수)
4. 프로젝트 디렉토리에서 오늘 날짜의 git 커밋 이력 수집
5. `C:\Users\EliteDesk\.claude\session-logs\YYYY-MM-DD.md` 에 누적 추가

### settings.json 등록
**경로**: `C:\Users\EliteDesk\.claude\settings.json`

```json
{
  "hooks": {
    "SessionEnd": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "powershell -NoProfile -ExecutionPolicy Bypass -File C:\\Users\\EliteDesk\\.claude\\hooks\\session-summary.ps1"
          }
        ]
      }
    ]
  }
}
```

## 생성되는 로그 예시

**파일**: `C:\Users\EliteDesk\.claude\session-logs\2026-04-25.md`

```markdown
# Claude Code 작업 일지 - 2026-04-25

## 14:32 | koogAgent_H
**요청 목록**
- AgentMemory 개발해
- memory add 내 이름은 hero 야 하면...
- commit 해줘

**사용 도구**: Edit(12), Bash(8), Read(6), Write(3)

**오늘 커밋**
```

하루에 여러 세션을 열어도 같은 날짜 파일에 누적 추가된다.

## 참고: Stop 훅과의 차이

| | `Stop` | `SessionEnd` |
|---|---|---|
| 발동 횟수 | 매 응답 턴마다 | 세션 종료 시 1회 |
| 용도 | 턴 단위 검증, 빌드 체크 등 | 세션 전체 요약, 정리 |
| 무한루프 주의 | `stop_hook_active` 필드로 감지 | 해당 없음 |
