# 24. RAG - 코드베이스 의미 기반 검색 구현

## 배경

현재 `CodeSearchTool`은 `ripgrep` 키워드 매칭 방식이라 정확한 단어를 알아야 검색이 가능하다.
RAG를 적용해 **의미 기반 검색**을 추가하면 "이 메서드가 하는 일과 비슷한 코드 찾아줘" 같은
자연어 질문으로도 관련 코드를 찾을 수 있다.

```
기존: CodeSearchTool  →  rg "keyword"  →  키워드 매칭
추가: RagSearchTool   →  임베딩 유사도  →  의미 기반 검색
```

---

## 구현 전략

- 임베딩 API: **OpenAI `text-embedding-3-small`** (`.env`의 `OPENAI_API_KEY`)
- 벡터 저장소: **파일 기반 JSONL** (`.koogagent/rag/` 디렉토리) — 외부 서비스 불필요
- 청크 단위: **메서드/클래스 단위** (파일 전체보다 작고 라인보다 맥락이 풍부)
- 유사도: **코사인 유사도**

```
.koogagent/
└── rag/
    ├── index.jsonl       ← 청크 메타데이터 + 임베딩 벡터
    └── meta.json         ← 마지막 인덱싱 시각, 파일 해시
```

### Step 6: RagSearchTool 구현 (TDD)

**파일**: `src/main/java/koogagent/tools/RagSearchTool.java`

```java
package koogagent.tools;

import ai.koog.agents.core.tools.annotations.LLMDescription;
import ai.koog.agents.core.tools.annotations.Tool;
import ai.koog.agents.core.tools.reflect.ToolSet;
import koogagent.rag.EmbeddingClient;
import koogagent.rag.VectorStore;

public class RagSearchTool implements ToolSet {

    private final EmbeddingClient embeddingClient;
    private final VectorStore vectorStore;
    private static final int TOP_K = 5;

    public RagSearchTool(EmbeddingClient embeddingClient, VectorStore vectorStore) { ... }

    @Tool(customName = "ragSearch")
    @LLMDescription("""
        코드베이스에서 의미 기반으로 관련 코드를 검색합니다.
        키워드 정확도보다 의미·맥락이 중요할 때 사용하세요.
        예: "HTTP 요청 처리하는 코드", "에러 핸들링 패턴", "인증 관련 로직"
        상위 5개 관련 코드 청크를 파일 경로·라인·내용과 함께 반환합니다.
        """)
    public String ragSearch(
        @LLMDescription("찾고자 하는 코드의 설명 (자연어)") String query) throws Exception { ... }
}
```

출력 형식:
```
[1] src/main/java/koogagent/CodingAgent.java (54-73줄) [유사도: 0.87]
public String chat(String userMessage) throws Exception {
    ...
}

[2] ...
```

**테스트**: `src/test/java/koogagent/tools/RagSearchToolTest.java`

테스트 케이스:
- `ragSearch_returnsTopK_formatted()` — Mock VectorStore/EmbeddingClient로 포맷 검증
- `ragSearch_returnsNoResult_message_whenEmpty()`

### Step 7: CodingAgent 통합

**파일**: `src/main/java/koogagent/CodingAgent.java`

변경 사항:
1. `RagSearchTool`을 `ToolRegistry`에 추가
2. 생성자에서 `OpenAiEmbeddingClient` + `FileVectorStore` + `RagSearchTool` 초기화
3. 시작 시 `CodebaseIndexer.indexIncremental(".")` 호출 (비동기)

```java
// ToolRegistry에 추가
.tools(new RagSearchTool(embeddingClient, vectorStore))

// 생성자에서 비동기 인덱싱 시작
CompletableFuture.runAsync(() -> indexer.indexIncremental(Path.of(".")));
```

`.env`에 추가:
```
OPENAI_API_KEY=sk-...
RAG_ENABLED=true   # false이면 RagSearchTool 미등록
```
